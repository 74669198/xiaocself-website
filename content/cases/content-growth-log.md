## 写在前面

这个网站最初只有 7 篇文章，是我用 Claude Code 搭建网站时顺手写的。内容少、结构散，Google 搜索根本找不到。

我给自己定了个目标：**一个月内把内容扩充到 20 篇**。

这篇文章记录我如何用 Claude Code 辅助内容创作，从选题到发布，以及中间的 SEO 优化和数据追踪。

**问题**：

- "关于"分类空白，网站没有信任感
- 没有针对 SEO 的关键词规划

### 竞品分析

我让 Claude 帮我分析同类技术博客的内容结构：

```
> 分析一下 Claude Code 相关技术博客的内容结构，哪些话题最受欢迎？
```

Claude 的分析结论：
1. **配置指南类**搜索量最大，因为新手最多
2. **实战案例类**粘性最高，读者会反复来看
3. **工具对比类**分享率最高，容易获得外链
4. **个人故事类**建立信任，转化率最高

这直接影响了我的选题策略。

4. **案例类用真实数据**——增加可信度

## 第三阶段：写作流水线

### Claude 辅助写作的 4 步法

**Step 1：给 Claude 大纲**

```
> 帮我写一篇"Claude Code Git 工作流最佳实践"，大纲如下：
> 1. 分支策略（Feature Branch）
> 2. 提交信息规范化（Conventional Commits）
> 3. 代码审查
> 4. 冲突解决
> 5. 回滚操作
> 6. 完整工作流演示
> 风格：口语化、有代码示例、像跟同事聊天
```

**Step 2：Claude 生成初稿**

Claude 会在 30 秒内生成 2000+ 字的初稿。

**Step 3：人工审查修改**

我主要修改：
- 补充真实案例和数据
- 调整语气和风格
- 确保技术准确性
- 添加个人经验

**Step 4：SEO 优化**

```
> 帮我优化这篇文章的 SEO：
> - 确保关键词出现在标题、描述、H2 标签
> - 添加内链到相关文章
> - 优化 meta description
```

### 效率对比

| 环节 | 纯手写 | Claude 辅助 | 提升比例 |
|------|--------|------------|---------|
| 大纲设计 | 30 分钟 | 5 分钟 | 83% |
| 初稿撰写 | 3-4 小时 | 30 分钟 | 85% |
| 修改润色 | 1 小时 | 1 小时 | 0% |
| SEO 优化 | 30 分钟 | 10 分钟 | 67% |
| **总计** | **5-6 小时** | **1.5-2 小时** | **~70%** |

关键发现：**Claude 省的是"从 0 到 1"的时间，"从 1 到好"还是需要人工**。

## 第四阶段：发布节奏

### 发布策略

我采用了"先发后改"的策略：

1. **初稿直接发布**——确保内容及时上线
2. **观察 3 天数据**——哪些文章有搜索流量
3. **根据数据优化**——有流量的文章重点打磨

### 发布时间表

| 日期 | 发布文章 | 累计 |
|------|---------|------|
| Day 1-2 | 入门指南、CC-Switch、Hooks、使用技巧 | 4 篇 |
| Day 3 | 微信机器人、建站实录 | 6 篇 |
| Day 4 | Hugo 速查表、SEO 指南 | 8 篇 |
| Day 5-7 | Superpowers 工作流、我的故事 | 10 篇 |
| Day 8-10 | Settings 指南 | 11 篇 |
| Day 11-14 | Git 工作流、多项目管理、MCP 指南、AI 对比 | 15 篇 |

### 前期集中发布的原因

Google 对新站有"沙盒期"，内容越多越早被索引。前 3 天发了 8 篇，确保 Google 知道这是个活跃的站点。

## 经验总结

### 做对了的事

1. **选题矩阵**——有规划地覆盖关键词，不盲目写
2. **Claude 辅助写作**——效率提升 70%，省下的时间做优化
3. **先发后改**——及时上线比完美更重要
4. **SEO 基础先行**——sitemap、meta、结构化数据一次性配好

### 可以改进的

1. **应该更早提交 Search Console**——延迟了 1 周，影响索引速度
2. **图片优化不足**——OG 图片没做好，影响社交分享
3. **内链策略太弱**——文章之间关联不够

### 给新站的建议

1. **内容 > 技术**——20 篇好内容比 5 篇完美内容强
2. **Claude 写初稿，你写灵魂**——AI 负责结构，你负责个性和经验
3. **SEO 是长期游戏**——前 30 天的数据不代表最终结果
4. **坚持发布节奏**——Google 偏好活跃更新的站点
