## 工作流一：晨会信息处理（Notion AI + Kimi）

**场景**：每天早上 9 点，各部门在群里发昨日数据和工作汇报，我需要整理成一份晨会简报。

**以前的流程**：
1. 打开 5 个微信群，复制关键信息
2. 粘贴到 Word，手动排版
3. 整理格式，发邮件给管理层
4. **耗时：45 分钟**

**现在的流程**：
1. 把群里的汇报截图/文字发给 Kimi
2. 指令："整理成表格，包含部门、昨日完成、今日计划、需协调事项四列"
3. Kimi 输出 Markdown 表格
4. 粘贴到 Notion，Notion AI 自动优化排版
5. **耗时：8 分钟**

**关键技巧**：
- 给 Kimi 的指令要包含输出格式（表格/列表/段落）
- Notion AI 的「优化排版」比手动调快 5 倍
- 把最终模板存为 Notion 模板，每天复制一份

## 工作流二：工资数据核对（Claude Code + Excel）

**场景**：每月 5 号核对上月工资，30+ 人的数据，涉及基本工资、绩效、扣款、加班费。

**以前的流程**：
1. 财务发 Excel 表
2. 打开表格，逐项核对公式
3. 发现问题标黄，发回财务修改
4. 来回 3-4 轮
5. **耗时：4-6 小时**

**现在的流程**：
1. 财务发 Excel 表
2. 用 Claude Code 读取文件：
   ```
   读取 salary_202605.xlsx，核对以下规则：
   - 绩效工资 = 基本工资 × 30% × (绩效评分/100)
   - 迟到扣款：3次以内不扣，超过3次每次50
   - 请假扣款 = 请假天数 × (基本工资/21.75)
   - 加班费 = 加班小时 × (基本工资/21.75/8) × 1.5
   找出所有不符合规则的行
   ```
3. Claude 生成核对脚本，自动标记异常
4. **耗时：15 分钟**

**关键技巧**：
- 把核对规则写成 Claude 的配置文件（`~/.claude/settings.json` 里的 env），下次直接调用
- 异常数据导出为 Excel，标红发给财务，对方改起来也清楚
- 脚本可以复用，每月只需改文件名

## 工作流三：培训材料制作（Kimi + Midjourney）

**场景**：给团队做新员工培训，需要 PPT + 手册 + 测试题。

**分工**：
- **Kimi**：写大纲、生成测试题、写手册文字
- **Midjourney**：做 PPT 封面、章节配图
- **我**：审校内容、调整风格

**具体步骤**：

1. **大纲生成**（Kimi，5 分钟）
   ```
   我是运营团队负责人，要给新员工做入职培训。
   主题是"客户沟通规范"，受众是 20-30 岁的一线运营人员。
   生成一份培训大纲，包含：
   - 5 个章节
   - 每个章节 3 个要点
   - 每个要点配一个真实案例
   ```

2. **PPT 配图**（Midjourney，20 分钟）
   根据大纲的每个章节，生成一张配图：
   ```
   professional business illustration, customer service team collaboration,
   clean modern style, soft blue and white color palette --ar 16:9
   ```

3. **测试题**（Kimi，3 分钟）
   ```
   根据上面的培训大纲，生成 10 道单选题 + 2 道案例分析题。
   附答案和解析。
   ```

4. **手册排版**（我，30 分钟）
   把 Kimi 生成的内容放进 Word/PPT，调整格式。

**总耗时：约 1 小时**（以前需要 1 天）

## 工作流四：客户咨询自动回复（OpenClaw + Kimi）

**场景**：团队微信群里经常有重复咨询（"工资什么时候发""怎么请假""报销流程是什么"）。

**方案**：OpenClaw 微信机器人 + Kimi 模型自动回复。

**配置要点**：
- 只在工作时段（9:00-18:00）自动回复
- 非工作时间回复："工作时间 9:00-18:00，紧急事项请拨打 XXX"
- 设置白名单，只对内部群生效
- 设置「兜底回复」：当 AI 不确定时，回复「这个问题我需要确认一下，稍后回复您」

**效果**：
- 80% 的常见问题由 AI 自动处理
- 复杂问题转人工，响应时间从平均 2 小时降到 15 分钟
- 员工满意度从 72% 提升到 89%

## 工作流五：周报自动生成（Claude Code + Notion）

**场景**：每周五要写周报，汇总本周工作。

**以前**：翻聊天记录、看邮件、回忆做了什么——耗时 1 小时，还漏事项。

**现在**：
1. Claude Code 读取本周的 git commit 记录
2. 读取本周修改过的文件
3. 指令：
   ```
   根据本周的代码提交和文件修改，生成一份工作周报：
   - 已完成的工作（3-5 条）
   - 遇到的问题和解决方式（1-2 条）
   - 下周计划（2-3 条）
   用第一人称，简洁明了
   ```
4. 粘贴到 Notion，微调后提交

**耗时：10 分钟**

## 工具选择的思考

**为什么不全用 Claude Code？**

Claude Code 很强，但不是万能的：

| 场景 | 为什么不用 Claude Code | 替代工具 |
|------|----------------------|---------|
| 长文本阅读（10万字+）| 上下文不够长 | Kimi（支持 200K）|
| 图像生成 | 没有图像能力 | Midjourney |
| 微信消息收发 | 需要网关中间件 | OpenClaw |
| 结构化笔记 | 没有笔记管理功能 | Notion |

**核心原则：让工具做它擅长的事，不要硬拧。**

## 给你的建议

如果你是刚开始用 AI 工具，不要一次上太多。建议按这个顺序：

**第 1 周**：选一个最痛的场景，用一个工具解决（推荐 Claude Code 处理文档/数据）
**第 2-4 周**：固化流程，形成习惯
**第 2 个月**：增加第二个工具，解决第二个痛点
**第 3 个月**：考虑工具之间的协作

**不要做的**：
- 为了用工具而用工具（先解决问题，再考虑工具）
- 一次性买齐所有会员（先用免费额度验证需求）
- 让 AI 完全替代人（AI 是助手，决策权在人）
